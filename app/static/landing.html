Za koga je namenjeno

Proizvod je namenjen timovima koji pre slanja ili obrade teksta moraju da uklone osetljive podatke i zadrže kontrolu nad sadržajem.

Pravne službe

Redakcija identifikatora pre slanja ugovora, dopisa, izjava, zahteva i druge dokumentacije trećim stranama.

Računovodstvo i finansije

Maskiranje PIB-a, matičnog broja, IBAN-a i brojeva kartica pre deljenja sadržaja sa eksternim sistemima i partnerima.

AI i automatizacija

Čišćenje teksta pre slanja ka LLM alatima, internim AI tokovima, ticketing sistemima i automatizovanim pipeline-ovima.

Šta rešava

Umesto ručnog traženja i zamračivanja podataka, servis radi standardizovanu redakciju kroz jednu API tačku.

Tipični problemi

  • Dokumenti se dele bez dosledne provere osetljivih podataka.
  • Timovi ručno brišu JMBG, PIB, email ili telefone iz teksta.
  • Pre slanja podataka AI sistemima ne postoji standardni privacy sloj.
  • Nema jedinstvenog načina za maskiranje, hash ili uklanjanje vrednosti.

Rezultat

  • Brža priprema teksta za deljenje i obradu.
  • Manje rizika od slučajnog otkrivanja podataka.
  • Jedinstven API za više tipova osetljivih informacija.
  • Jasan demo i API dokumentacija spremni za integraciju.

Kako radi

Demo i API rade nad istim backend-om. Tok je jednostavan i spreman za internu upotrebu ili integraciju.

1

Unos teksta

Korisnik ili sistem šalje tekst koji treba očistiti pre deljenja ili dalje obrade.

2

Detekcija entiteta

Servis prepoznaje podržane tipove kao što su JMBG, PIB, IBAN, email, telefon i drugi.

3

Primena politike

Za svaki pronađeni entitet primenjuje se maskiranje, hash zamena ili potpuno uklanjanje.

4

Povrat rezultata

API vraća redigovani tekst, detections listu i audit metapodatke bez čuvanja sirovog teksta.
