Šta proizvod radi

API prima tekst, prepoznaje osetljive entitete i vraća redigovanu verziju kroz standardne politike: mask, hash ili rm.

Za koga je namenjen

Za pravne, finansijske i AI timove koji moraju da zaštite podatke pre slanja sadržaja ka eksternim sistemima.

Podržani tipovi osetljivih podataka

JMBG PIB MB_COMPANY EMAIL PHONE IBAN CARD_NUMBER

Koraci postavljanja za novog klijenta

1

Kickoff

Definišemo use-case, očekivani obim i kriterijume uspeha.

2

API ključ

Kreiramo klijenta i dodeljujemo API ključ kroz postojeći admin tok.

3

Integracija

Klijent povezuje svoj servis na endpoint za redakciju.

4

Validacija

Testiramo realne uzorke i usaglašavamo pravila redakcije.

5

Pilot

Pokreće se vremenski ograničen pilot sa praćenjem rezultata.

Kako se koriste API ključevi

  • Svaki klijent ima jedinstven ključ.
  • Ključ se šalje kroz `x-api-key` zaglavlje.
  • Ključ je vezan za status, rate limit i dozvoljene entitete.
  • Rotacija ključa je podržana kroz admin rutu.

Kako pilot funkcioniše

  • Pilot traje dogovoreni period i ciljano pokriva jedan proces.
  • Meri se broj obrađenih zahteva i kvalitet redakcije.
  • Rezultat pilota je preporuka za sledeći komercijalni korak.
