Onboarding checklist (interno)

  • Identifikuj use-case i vlasnika procesa
  • Potvrdi tipove entiteta i politiku (mask/hash/rm)
  • Kreiraj klijenta i dodeli plan
  • Postavi limite (RPM, quota) i status
  • Pošalji API ključ i tehničke smernice
  • Potvrdi testne primere (real samples, sanitized)
  • Prati usage i incident/feedback signal
  • Završi pilot sa zaključkom i sledećim korakom

Šta šaljemo klijentu

  • Linkovi (landing/offer/onboarding/demo/docs)
  • API ključ (x-api-key) i osnovne bezbednosne smernice
  • Dogovoreni entiteti + politika redakcije
  • Pilot okvir: obim, KPI i datum revizije
  • Kontakt tačka za tehnička pitanja

Standardni pilot success criteria

  • Integracija: prvi uspešan poziv iz klijentovog sistema
  • Kvalitet: pokrivenost dogovorenih entiteta u uzorcima
  • Operativno: stabilna latencija i bez grešaka u normalnom opterećenju
  • Bezbednost: API key čuvan server-side (ne u javnom frontendu)
  • Ishod: jasna odluka za production rollout ili korekcije

Standardni interni koraci (operativno)

U admin-u

  • Create client
  • Assign plan
  • Set limits (RPM, quota)
  • Confirm allowed entities
  • Send API key (one-time display)

Tokom pilota

  • Monitor usage summary
  • Review errors/feedback
  • Rotate key if needed
  • Update plan/limits after review
