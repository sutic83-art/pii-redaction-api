Politika privatnosti (sažeto)

Ova stranica objašnjava koje podatke servis obrađuje i koje evidencije se čuvaju u trenutnoj verziji.

Šta servis obrađuje

  • Tekst koji pošaljete na redakciju (API ili demo) se obrađuje da bi se prepoznali i redigovali osetljivi entiteti.
  • Servis vraća redigovan tekst i metapodatke o detekcijama (bez potrebe da čuva sirov tekst).
Napomena: U ovoj verziji cilj je da se sirov sadržaj ne zadržava duže nego što je potrebno za obradu zahteva.

Šta se čuva / loguje

  • Kontakt forma: prijave (ime, email, kompanija opcionalno, poruka) se čuvaju kao evidencija upita.
  • Konfiguracija klijenata: postoji model klijenta (plan, status, limiti, dozvoljeni entiteti) i admin rute za upravljanje.
  • Praćenje upotrebe: beleži se broj zahteva i osnovni metapodaci o upotrebi (npr. client_id, plan, ruta, broj detekcija, dužina teksta), radi kontrole i kapaciteta.

API ključevi i tajne

  • API ključ se koristi za autorizaciju pristupa po klijentu.
  • API ključ ne treba izlagati u javnom frontendu (npr. u browser aplikaciji) — integracije se rade server-side.
  • Admin ključ štiti admin rute i treba da bude čuvan kao tajna (env/secret store).
