Security (sažet pregled)

Ovo je realističan pregled trenutnih kontrola i operativnog modela u ovoj verziji proizvoda.

Ne tvrdimo sertifikate (npr. ISO/SOC) osim ako se eksplicitno ne dogovore i ne verifikuju u posebnom aranžmanu.

Kontrole pristupa

  • API ključ (x-api-key) po klijentu.
  • Status klijenta (active/disabled) i mogućnost rotacije ključa.
  • Admin rute za upravljanje su zaštićene admin ključem (x-admin-key).

Limiti i kontrola zloupotrebe

  • Per-client rate limit (zahtevi/minut).
  • Mesečne kvote (po potrebi) i evidencija upotrebe.
  • Ograničenje entiteta po klijentu (allowed_entities).

Redaction flow (bezbednosna logika)

  • Klijent šalje tekst server-side, uz API ključ.
  • Servis detektuje i rediguje osetljive entitete pre daljeg slanja ka eksternim sistemima.
  • Preporuka: ključ nikada ne izlagati u javnom browser frontendu.

Skladištenje podataka u ovoj verziji

  • Nema baze podataka po default-u u ovom projektu.
  • Postoje interne evidencije (kontakt upiti, usage tracking, konfiguracija klijenata) u file-based modelu.

Deploy i operacije

  • Produkcijski deploy model je baziran na standardnom FastAPI servisu.
  • CI/CD i standardni build/deploy tok (npr. Docker) su predviđeni u projektu.
  • Preporuka: tajne (admin key, API keys) držati u environment/secret store, ne u kodu.
